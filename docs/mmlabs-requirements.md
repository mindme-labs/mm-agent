Структура сервиса
Сервис состоит из следующих компонент:
лендинг с описанием сервиса, его возможностей и перехода к регистрации и авторизации в сервисе.
личный кабинет пользователя, где ему доступны функции сервиса
административная панель сервиса для управления его настройками
сервис анализа файлов с выгрузкой по счетам
интеграционный сервис для взаимодействия с 1С Бухгалтерией
интеграционный сервис для взаимодействия с поставщиком AI функций
сервис биллинга для начисления оплат
Процесс работы с сервисом
Для каждого пользователя предусмотрен следующий процесс работы с сервисом:
регистрация в сервисе - проверка доступности сервиса для регистрирующегося пользователя
определение режима работы сервиса для пользователя (демо/пре-прод/продакшен)
прохождение онбординга
переход к работе сервиса в режиме, настроенном по итогам онбординга.

Есть 3 режима работы для пользователей:
Демо: нет ИИ, нет данных из 1С, только демо-данные из 1С бухгалтерии, показывает как бы мог работать сервис наличии выгрузки из 1С и доступа к ИИ
Пре-прод: есть ИИ, есть выгрузка из 1С, нет постоянного соединения с 1С, показывает какую реальную аналитику может дать сервис на основе настоящих данных пользователя, поясняет как это будет работать в ежедневном режиме
Прод: есть ИИ, есть постоянное соединение с 1С, сервис работает в полноценном режиме


Сервис должен поддерживать 3 рабочих режима, которые настраиваются для каждого пользователя:
Демо:
Пользователь логиниться в систему
Сервис открывает экран онбординга с кратким описанием шагов онбординга для получения результата
Сервис открывает экран для загрузки excel или csv файлов с выгрузкой данных из 1С Бухгалтерии. В демо-режиме файлы подгружены заранее и пользователь видит это в интерфейсе
Сервис открывает экран “загрузка и анализ данных”
Сервис открывает экран “рекомендации по улучшению ведения учета”
Сервис открывает экран “Обнаруженные проблемы”. Проблемы отображаются в виде карточек, которые пользователь может проигнорировать, взять в работу, написать рекомендованные сообщения для ответственных лиц по предложенному тексту.
После выхода пользователя из системы результаты его сессии стираются и он все повторяет.
Демо с имитацией подключения к 1С


Пре-прод
Пользователь логиниться в систему
Сервис открывает экран онбординга с кратким описанием шагов онбординга для получения результата
Сервис открывает экран для загрузк excel или csv файлов с выгрузкой данных из 1С Бухгалтерии. Пользователь загружает не более 10 файлов, которые суммарно весят не более 10 Мб
Сервис открывает экран “загрузка и анализ данных”
Сервис открывает экран “рекомендации по улучшению ведения учета”
Сервис открывает экран “Обнаруженные проблемы”. Проблемы отображаются в виде карточек, которые пользователь может проигнорировать, взять в работу, написать рекомендованные сообщения для ответственных лиц по предложенному тексту.
В системе сохраняются результаты действий пользователя и карточки, сформированные на предыдущем шаге
В системе сохраняется информация об обращениях к ИИ сервисам для учета расхода токенов
Продакшен
Пользователь логиниться в систему
Сервис открывает экран онбординга с кратким описанием шагов онбординга для получения результата
Сервис открывает экран для назначения встречи по подключению к 1С Бухгалтерии.
Здесь онбординг прерывается до момента подключения. Как только система подключена и получены первые данные из 1С Бухгалтерии, пользователю приходит уведомление с просьбой вернуться в приложение.
Сервис открывает экран “загрузка и анализ данных”
Сервис открывает экран “рекомендации по улучшению ведения учета”
Сервис открывает экран “Обнаруженные проблемы”. Проблемы отображаются в виде карточек, которые пользователь может проигнорировать, взять в работу, написать рекомендованные сообщения для ответственных лиц по предложенному тексту.
В системе сохраняются результаты действий пользователя и карточки, сформированные на предыдущем шаге
В системе сохраняется информация об обращениях к ИИ сервисам для учета расхода токенов

Процесс работы демо-режима
пользователь авторизуется в системе через учетную запись Google
сервис проверяет наличие информации о пользователе, если пользователь уже есть, то определяет режим работы сервиса для данного пользователя, если нет, то показывает ошибку “для доступа к сервису вам надо получить приглашение от одного из пользователей”
пользователь выбирает использовать демо данные или вводит свои данные для регистрации (выбор ИП/Юр лицо, название юр лица, ИНН, свои ФИО)
пользователю отображается интерфейс с подставленными демо-файлами с выгрузками по разным счетам (задача: взять демо файлы из демо базы 1С Бух)
для демо режима в системе должны быть сохранены демо-данные с рекомендациями об оптимизации процесса ведения учета в 1С Бух и рекомендациями на основе данных из выгрузок
рекомендации формулируются в виде задач, где есть заголовок (в чем проблема), краткое описание сути проблемы и текст для отправки сообщения ответственному лицу (в зависимости от сути проблемы и зоны ответственности)
пользователь может выполнять следующие действия с рекомендациями:
копировать текст рекомендуемого сообщения
откладывать карточки для последующей работы
закрывать карточки если считает их нерелевантными (отдельная опция, может стоит назвать “отклонить”)
закрывать карточки если считает их выполненными
пользователь должен видеть ключевые метрики бизнеса на основе загруженных файлов из бухгалтерии (оборот за период, кредиторская задолженность, дебиторская задолженность, количество рекомендаций, требующих внимания)
